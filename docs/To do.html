User Story Basic Backlog

As a user I can:
  • browse through the places 
  • filter places by: location/cuisine/rating
  • add a place to favorites
  • book a table and choose which one (seat map)
  • add my name on a waiting list
  • see my reservation on a calendar
  • get notification before reservation
  • get the location of the place

As an administrator I can:
  • add places
  • add photos/menu/schedule
  • see the booked tables
  • do everything a user can

Must Features:
  • List of places
  • Schedule
  • Calendar
  • Favorite List
  • Seat Map 
  • Making a reservation
  • Authentification

Backend Backlog
  • save & fetch data from web
  • won`t directly access the database -> will be insecure

Communicate with a Server
  • won`t access database directly -> will be insecure
    • e.g. would have to store credentials for accessing the database in our Ionic app code, which is JS and people could get access to it
  • connect to an API exposed on a server (REST API)
    • accepts Https requests -> handles them -> run some code on the server and communicate with it
    • server interacts with database(which also runs on the server which also runs in the cloud)
    • only send requests + receive responses -> exchanged in JSON format
    • use a Service -> Firebase

  • + Wordpress REST API for table booking(found wordpress plugin - hope it`ll work)

Add Google Maps + Photos
  • use an API provided by google that translates coordinate into a real address to display for humans and use the Google Maps static image API to display a preview of the location on the screen
    • setting up the Google Maps API
    • Add Google Maps + Geocoding
  • Capacitor APIs used: 
    • geolocation -> get the current location of the user
    • camera -> use the device camera to take pics and upload them

Capacitor
  • Ionic app -> web app practically wrapped in a native app provided by Capacitor
    • in order to not use native API`s for different platform it provides a bridge to them -> JavaScript methods to call which basically translates to native APIs calls that allow us to use them
  • gives a rich set of build-in APIs we can get through 1 JS API and with one method (no matter where out app runs -iOS/Android/web-) we reach the native APIs

Authentification
  • Facebook plugin support for Capacitor -found (released last or this month, only thing available yet)
  • everything happens in the browser, not re-fetch HTML pages
  •  send auth token
